# Support

## Getting Help

- **Bug reports** — Open an issue in the relevant repository
- **Feature requests** — Open an issue describing your idea
- **Questions** — Use the Discussions tab if available, or open an issue

## Before Opening an Issue

1. Search existing issues to avoid duplicates
2. Check the README for setup instructions and known limitations
3. Include reproduction steps and environment details

## Response Times

These are personal/side projects maintained in my spare time. I'll do my best to respond promptly, but there are no guaranteed SLAs.
